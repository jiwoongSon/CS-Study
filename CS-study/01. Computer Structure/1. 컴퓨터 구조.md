
# 컴퓨터의 역사
- MARK-I (1944): 최초의 전기 기계식 계산기. **천공카드**
	- 천공카드의 구멍: 구멍이 있어 전기가 통함 -> 1로 인식
- ENIAC (1946): 범용 전자식 컴퓨터, 1만 8천개의 **진공관**을 사용(속도의 비약적인 향상), 10진법
	- 진공 상태의 유리관: 음극에 열을 가하면 전자들이 나옴-> 양극으로 날아가려고 함. 
		- 이때 중간에 있는 Grid에 어떤 전압을 걸어주냐에 따라, 전자를 통과시킬수도, 튕겨낼 수 도 있다. 하지만 열이 너무 
- EDVAC (1949): **폰 노이만 구조**, 2진법 사용, 설계만 에드삭보다 먼저 됨.
- EDSAC (1949): 폰 노이만 구조 방식을 최초 실제로 가동
- UNIVAC-I (1951): 세계 최초의 상업용 컴퓨터

# Von Neumann Architecture
- 이전의 컴퓨터는, 하나의 Hardware 당 하나의 프로그램만 실행 할 수 있었다. ENIAC의 경우 새로운 프로그램(계산)을 실행할 때 마다 직접 전선(진공관, 천공카드)을 바꾸어 새로운 Hardware로 만들어야 했다. 
- Software, Data를 모두 Memory라는 새로운 창고에 도입하고, CPU가 이를 순서대로 꺼내서 실행하자. 
	- CPU, Memory, I/O 로의 분리

## 폰 노이만 병목현상 
- CPU의 연산은 빠르다. 하지만 Memory의 속도는 느리고, Bus도 하나뿐이라 Instruction이나 Data를 가져올 때 마다 병목현상이 발생한다. 
	- Static RAM: CPU 내부에서 사용, Filp-Flop Logic gate를 사용하여 데이터를 저장한다. 
	- Dynamic RAM: Capacitor를 사용하여 데이터를 저장, Capacitor는 시간이 지나면 전자가 방전된다. 따라서 데이터를 유지하려면 주기적으로 전기를 Refresh 해줘야 하는데, Refresh를 하는 동안에는 CPU가 Memory에서 데이터를 읽거나 쓸 수 없다. 따라서 시간 지연이 일어난다. 
	- 또한 절대적인 물리 거리에 의해서 속도 차이가 나기도 한다. 
- CPU 내부에 메모리와 속도 차이를 줄여주는 Cache Memory 도입. 
## Filp-Flop
- 1bit의 정보를 저장할 수 있는 단위 기억 소자. 
- 전원이 공급되는 한 현재의 상태를 계속 유지한다. 즉 기억 능력이 있다. 
- Sequential Logic Curcuit: 과거의 상태가 영향을 미침. 
- Clock Pulse에 맞추어, 상태를 변경한다.
	- 클럭 펄스에 변화가 있을 때 -> 너의 핀에 어떤 전압이 걸려있는지 확인해라. 라는 명령
	- 핀에 걸려있는 전압은 연산 결과에 의해 미리 결정되어 있는 상태이다. 

- Set-Reset 플립플롭
	- Set: 출력을 1로 만듦.
	- Reset: 출력을 0으로 만듦. 
	- S=1, R=1가 동시에 입력되는 경우 Undefined

- Data/Delay 플립플롭
	- SR플립플롭 단점 해결, 입력선을 하나로 줄임
	- 입력된 Data를 클럭 신호에 맞추어 그대로 출력으로 내보냄. 
	- 잠시 Delay 시키는 역할. 
	- 현재의 Register와 SRAM에 자주 사용

- JK 플립플롭
	- J는 Set, k는 Reset의 역할 
	- J=1, K=1 이 동시에 입력되는 경우, 현재 상태를 반대로 뒤집는 Toggle 동작 수행
	- 모든 플립플롭의 기능을 구현할 수 있다. 

- T 플립플롭 
	- JK 플립플롭의 2개의 입력선을 하나로 묶음.
	- T=0 이면 현재 상태 유지, T=1 인 경우 Toggle
	- Couter 회로를 설계할 때 주로 사용

# CPU 내부 구조
- ALU: Arithmetic Logic Unit
	- Full Adder가 아주 촘촘하게 엮여 있다. 물리적인 연산이 이루어지는 곳
	- 산술연산, 논리연산 수행 

- CU: Control Unit
	- Memory에서 가져온 Instruction을 Decode
	- CPU 내부 부품들에게 Control Signal 발생시킴

- Register
	- CPU내부의 아주 작고 빠른 임시 기억장치. 
	- Program Counter : 다음에 실행할 명령어의 실제 메모리 주소 기억 
	- Instruction Register: 현재 실행 중인 명령어 자체를 보관 
	- Memory Address Register: CPU가 메모리의 특정 위치로 접근하는 경우, 해당 위치의 Address를 저장
	- Memory Buffer Register: 메모리에서 데이터를 가져오거나, 보낼 때 해당 데이터 자체가 잠시 머무른다. 
	- Accumulator: ALU가 연산할 데이터, 결과값 임시 저장

# Instruction Cycle

1. **인출 (Fetch):** "메모리에서 명령어를 가져오자"  
    • **PC**가 다음 명령어 주소를 알려줍니다.  
    • 해당 주소를 **MAR**을 거쳐 메모리로 보냅니다.  
    • 메모리는 해당 주소에 있는 명령어 코드를 **MBR**로 보내고, 이 코드는 최종적으로 **IR**에 저장됩니다. (동시에 PC는 다음 주소로 1 증가) 

2. **해독 (Decode):** "무엇을 하라는 건지 파악하자"  
    • **제어장치(CU)**가 **IR**에 있는 명령어를 쪼개어 분석합니다. "아하, 메모리에 있는 ‭A값을 가져와서 더하라는(ADD) 뜻이구나!"  

3. **실행 (Execute):** "가산기를 돌려라"  
    • 제어장치의 신호에 따라 연산에 필요한 데이터(‭A와 B)를 레지스터로 가져옵니다.  
	• 데이터가 **ALU**로 들어가 전가산기들을 통과하며 덧셈이 수행됩니다.  
	• 결과값은 **누산기(AC)**에 저장됩니다. 

4. **저장 (Store):** "결과를 남기자"  
    • 필요한 경우 누산기에 있는 결과값을 다시 메모리의 특정 주소로 보내어 저장합니다.
